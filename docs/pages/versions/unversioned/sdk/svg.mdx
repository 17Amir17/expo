### `Svg`

A set of drawing primitives such as `Circle`, `Rect`, `Path`,
`ClipPath`, and `Polygon`. It supports most SVG elements and properties.
The implementation is provided by [react-native-svg](https://github.com/react-native-community/react-native-svg), and documentation is provided in that repository.

<SnackInline label='SVG' dependencies={['react-native-svg']}>

```tsx
import * as React from 'react';
import Svg, { Circle, Rect } from 'react-native-svg';

export default function SvgComponent(props) {
  return (
    <Svg height="50%" width="50%" viewBox="0 0 100 100" {...props}>
      <Circle cx="50" cy="50" r="45" stroke="blue" strokeWidth="2.5" fill="green" />
      <Rect x="15" y="15" width="70" height="70" stroke="red" strokeWidth="2" fill="yellow" />
    </Svg>
  );
}
```

</SnackInline>

### SVG Modules

> SVG Modules is an experimental feature in SDK 50.

Expo CLI supports importing SVG files directly as React components by naming the SVG file with a `.module.svg` extension. For example, if you have a file named `logo.module.svg` in your project, you can import it like so:

#### Setup SVG Modules

<Step label="1">
Install **react-native-svg**

<Terminal cmd={['$ npx expo install react-native-svg']} />
</Step>

<Step label="2">
Create or modify the root **metro.config.js** file to enable the **isSVGEnabled** option:

```js metro.config.js
const { getDefaultConfig } = require('expo/metro-config');

/** @type {import('expo/metro-config').MetroConfig} */
const config = getDefaultConfig(__dirname, {
  /* @info Enable SVG Modules */
  isSVGEnabled: true,
  /* @end */
});

module.exports = config;
```

</Step>

Expo CLI uses [`SVGR`](https://github.com/gregberge/svgr) to convert SVG files to React components. You can customize the output by creating a [`svgr.config.js`](https://react-svgr.com/docs/configuration-files/) file in your project root. See the [SVGR documentation](https://react-svgr.com/docs/options/) for more information. This config is currently applied to all platforms and there is no way to set platform-specific options, this is partially why the feature is marked experimental.

- SVG files that don't have the `.module.svg` extension will be imported as standard assets (of type `number`), which can be rendered with Expo Image.
- SVG Modules support platform extensions: `home.module.native.svg`, `home.module.ios.svg`, `home.module.android.svg`, `home.module.web.svg`, etc. Be sure to import without the `.svg` extension to use.
- Ensure you don't pass `svg` to `sourceExts` in your `metro.config.js` file.
- Expo CLI's [typed environment](/router/reference/typed-routes#global-types) will automatically add TypeScript support for your SVG Modules.
- Scaled filename suffixes like `@1x`, `@2x`, `@3x` are not supported. Use `width` and `height` props to scale your SVG.

#### Migrating from `react-native-svg-transformer`

Unlike `react-native-svg-transformer`, SVG Modules enables you to use both asset SVGs and React SVGs simultaneously. If you want to use SVG Modules, you should disable `react-native-svg-transformer` and remove it from your `metro.config.js` file.

```diff metro.config.js
const { getDefaultConfig } = require('expo/metro-config');

/** @type {import('expo/metro-config').MetroConfig} */
const config = getDefaultConfig(__dirname, {
  // Enable SVG Modules support.
+  isSVGEnabled: true,
});

config.transformer = {
  ...config.transformer,
-  babelTransformerPath: require.resolve('react-native-svg-transformer'),
}

config.resolver = {
  ...config.resolver,
-  assetExts: assetExts.filter(ext => ext !== 'svg'),
-  sourceExts: [...sourceExts, 'svg'],
}

module.exports = config;
```

You can also uninstall the package:

<Terminal cmd={['$ yarn remove react-native-svg-transformer']} />

### Pro Tips

Here are some helpful links that will get you moving fast!

- Looking for SVGs? Try the [noun project](https://thenounproject.com/).
- Create or modify your own SVGs for free using [Figma](https://www.figma.com/).
- Optimize your SVG with [SVGOMG](https://jakearchibald.github.io/svgomg/). This will make the code smaller and easier to work with. Be sure not to remove the `viewbox` for best results on Android.
